## Workflow Secret Names Fix

### Problem
The GitHub Actions workflow was failing with:
```
Error: Error: Input required and not supplied: server
```

### Root Cause
The workflow was looking for `FTP_HOST`, `FTP_USERNAME`, `FTP_PASSWORD` secrets, but the GitHub repository has `SFTP_HOST`, `SFTP_USERNAME`, `SFTP_PASSWORD` configured.

### Solution
Updated `.github/workflows/scrape-and-deploy.yml` to use the correct secret names:
- `secrets.FTP_HOST` → `secrets.SFTP_HOST`
- `secrets.FTP_USERNAME` → `secrets.SFTP_USERNAME`  
- `secrets.FTP_PASSWORD` → `secrets.SFTP_PASSWORD`

- `745d6cb` - "fix: use SFTP_* secret names instead of FTP_*"
- `24e373e` - "docs: document workflow secrets and add warning comment"

## FTP Protocol Fix

### Problem
After fixing secret names, deployment failed with SSL errors when using FTPS.

### Solution
**ALWAYS USE PLAIN FTP** - The workflow must use:
- `protocol: ftp` (plain FTP, no encryption)
- `port: 21`
- Do NOT use `ftps`, `ftps-legacy`, or `sftp`

### Final Configuration
```yaml
- name: Deploy to Hostinger (FTP)
  uses: SamKirkland/FTP-Deploy-Action@v4.3.5
  with:
    server: ${{ secrets.SFTP_HOST }}
    username: ${{ secrets.SFTP_USERNAME }}
    password: ${{ secrets.SFTP_PASSWORD }}
    port: 21
    protocol: ftp
    local-dir: ./dist/
    dangerous-clean-slate: true
```

- `af3188c` - "fix: use plain FTP on port 21"

### IMPORTANT REMINDER
**Always use plain FTP protocol.** Do not change to SFTP or FTPS even if errors suggest SSL/TLS issues. The secret names are `SFTP_*` for historical reasons, but the protocol must remain plain `ftp` on port 21.
