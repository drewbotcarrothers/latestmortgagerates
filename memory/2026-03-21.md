## Workflow Secret Names Fix

### Problem
The GitHub Actions workflow was failing with:
```
Error: Error: Input required and not supplied: server
```

### Root Cause
The workflow was looking for `FTP_HOST`, `FTP_USERNAME`, `FTP_PASSWORD` secrets, but the GitHub repository has `SFTP_HOST`, `SFTP_USERNAME`, `SFTP_PASSWORD` configured.

### Solution
Updated `.github/workflows/scrape-and-deploy.yml` to use the correct secret names:
- `secrets.FTP_HOST` → `secrets.SFTP_HOST`
- `secrets.FTP_USERNAME` → `secrets.SFTP_USERNAME`  
- `secrets.FTP_PASSWORD` → `secrets.SFTP_PASSWORD`

- `24e373e` - "docs: document workflow secrets and add warning comment"

## SFTP Protocol Fix

### Problem
After fixing secret names, deployment failed with:
```
SSL routines:ssl3_get_record:wrong version number
```

### Root Cause
The workflow was using `ftps-legacy` protocol on port 21, but Hostinger requires **SFTP** (SSH-based) on port 22. FTPS and SFTP are different protocols.

### Solution
Updated `.github/workflows/scrape-and-deploy.yml`:
- Changed `protocol: ftps-legacy` → `protocol: sftp`
- Changed `port: 21` → `port: 22`
- Updated step name from "Deploy to Hostinger (FTP)" to "Deploy to Hostinger (SFTP)"
