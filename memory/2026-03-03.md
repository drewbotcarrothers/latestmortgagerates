## LatestMortgageRates.ca Deployment - Fixed

**Issues resolved:**
1. **SFTP → FTP migration**: Hostinger account uses FTP on port 21, not SFTP on port 65002. SSH public key auth was causing "Permission denied" errors.
2. **Workflow tool**: Switched from `lftp` (hung indefinitely) to `ncftpput` for reliable uploads.
3. **Directory path**: FTP user lands directly in `public_html`, so absolute path `/public_html` doesn't work - must use relative `.` or `public_html`.
4. **React hydration error #418**: Fixed by replacing `new Date()` calls in JSX with client-only components using `useEffect()` to render dates only after mount.

**Working workflow:**
- Tool: `ncftpput`
- Port: 21 (FTP)
- Path: relative `.` (current directory = public_html)
- Secrets mapping: `SFTP_USERNAME`, `SFTP_PASSWORD`, `SFTP_HOST` → env vars `FTP_USER`, `FTP_PASS`, `FTP_HOST`

**Next.js static export:**
- Output directory: `dist/`
- Contains: `index.html`, `404.html`, `_next/` (static assets)

**Outcome:** Site deployed successfully at https://latestmortgagerates.ca

## Plain Text Rendering Issue - FIXED ✅

**Root cause:** Missing Tailwind/PostCSS configuration files
- `tailwind.config.ts` didn't exist → Tailwind didn't know which files to scan
- `postcss.config.js` didn't exist → CSS wasn't being processed through Tailwind

**Solution:**
1. Created `postcss.config.js`:
```javascript
module.exports = {
  plugins: {
    tailwindcss: {},
    autoprefixer: {},
  },
}
```

2. Created `tailwind.config.ts`:
```typescript
import type { Config } from "tailwindcss";
const config: Config = {
  content: [
    "./pages/**/*.{js,ts,jsx,tsx,mdx}",
    "./components/**/*.{js,ts,jsx,tsx,mdx}",
    "./app/**/*.{js,ts,jsx,tsx,mdx}",
  ],
  theme: { extend: {} },
  plugins: [],
};
export default config;
```

**Before:** CSS was 190 bytes of raw `@tailwind` directives (uncompiled)
**After:** CSS compiled to thousands of lines with actual utility class definitions

**Status:** Site now renders with proper Tailwind styling at https://latestmortgagerates.ca
